Day 2 Development Documentation — Dental SaaS Backend
Overview

Today the backend architecture was upgraded from route-based logic to a controller-based architecture, making the system more modular, scalable, and aligned with industry practices.

The backend now follows a layered structure:

Routes → Controllers → Database

This separation improves maintainability and prepares the project for larger features and future scaling.

Multi-Tenant SaaS Logic

All resources are scoped using:

clinic_id

JWT payload structure:

{
 "clinic_id": 1
}

Middleware extracts the identity:

req.clinic.clinic_id

Every query filters data using:

WHERE clinic_id = req.clinic.clinic_id

This guarantees:

Clinic A cannot access Clinic B data
